Micro services are small, autonomous services that work together.

Something that could be rewritten in two weeks.

The characteristics thats make micro services different are :-

Small, and focused on doing one thing well

less  cohesive code

Robert C. Martin’s definition of “Single Responsibility Principle”(SRP) which states “Gather those things which changes for same reason, and separate those things which change for different reasons”. Micro services take this same approach to independent services.

Service boundaries are focussed on business boundaries making code to live for the functionality and thus avoiding the temptation for making service too large.

How small is small?

- getting idea from number of lines of code is problematic.

- as size decrease the benefits around interdependence increase but so does complexity of   many having parts.
